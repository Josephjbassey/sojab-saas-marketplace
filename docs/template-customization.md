# Template Customization Guide

Once a template is purchased, it can be customized to fit the specific needs of the customer.

## Customization Levels

1. **Theming:** Modifying Tailwind CSS configuration and brand assets.
2. **Configuration:** Updating settings via the template's `.env` or CMS-driven settings.
3. **Feature Toggle:** Enabling/disabling core modules (e.g., Blog, User Auth, API).
4. **Code-level Changes:** Extending the Django models and views within the generated project.

## Workflow

- **Manual Customization:** For MVP, customization requests are handled by the support team via `CustomizationRequest`.
- **Automated Customization (Future):** Using the Project Generator to apply transformations to the template source code.
